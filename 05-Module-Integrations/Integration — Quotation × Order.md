# Integration — Quotation × Order

#integration #module/order-mgmt #status/active

> How a Quotation's service catalog becomes an Order — snapshot copy, not live reference.

## Overview

A [[Quotation]] contains a catalog of [[Quotation Service]]s. When a customer places an order, the operator selects one or more services from that catalog. Each selected service becomes an [[Order Line]] with a copied [[Price Rule]] and [[Allocation Rule]] snapshot. From that point, the Order is self-contained.

## Data Flow

```
Quotation
  └── QuotationService (service + PriceRule + AllocationRule)
            │
            │  operator selects service(s) at order creation
            ▼
Order
  └── OrderLine (one per selected QuotationService)
        ├── PriceRule snapshot   ← copied from QuotationService
        ├── AllocationRule snapshot ← copied from QuotationService
        ├── service_id           ← for traceability
        ├── quotation_service_id ← for traceability
        └── OrderLocations       ← copied from Service's ServiceStops
              (adjustable before VRP)
```

## On Order Creation — Step by Step

1. Operator selects Customer → system loads Quotation
2. Operator picks N Quotation Services from catalog
3. For each selected QuotationService:
   a. Create [[Order Line]] linked to `quotation_service_id`
   b. Copy active [[Price Rule]] → new PriceRule scoped to OrderLine
   c. Copy active [[Allocation Rule]] → new AllocationRule scoped to OrderLine
   d. Copy [[Service Stop]]s → new [[Order Location]]s (address/geo snapshot from [[Customer Location]] master)
   e. Compute `OrderLine.revenue` from copied PriceRule
4. `Order.revenue = Σ OrderLine.revenue`
5. `OrderLocation.allocated_share` computed per AllocationRule (drops only)
6. Operator may adjust: PriceRule params, stop list, location overrides
7. Order confirmed → locked for VRP

## Price Adjustments at Order Time

Operator may modify copied PriceRule params before confirmation (e.g. negotiate a one-off discount):

```
OrderLine.price_manually_adjusted = true
OrderLine.revenue recomputed from adjusted params
Order.revenue recomputed
```

Adjustment is per-order, non-destructive. QuotationService PriceRule unchanged.

## Quotation Price Renegotiation

When customer renegotiates price for a service:

```
Close current QuotationService PriceRule (effective_to = today)
Create new PriceRule on QuotationService
Existing Orders with copied snapshots: unaffected
New Orders from same Quotation: use new PriceRule
```

Same principle as [[ADR-001 — Close and Replace, Never Mutate]].

## Traceability

```
Order.quotation_id            → which contract
OrderLine.quotation_service_id → which catalog entry
OrderLine.service_id           → which route template
```

Supports queries: "all orders from this quotation," "revenue generated by service X for customer Y."
